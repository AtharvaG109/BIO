## Problem

Raw packet analysis is powerful but hard to hand to another engineer or stakeholder. The useful version needed to connect capture, detection, diagrams, telemetry, and reporting without becoming a set of unrelated scripts.

## Architecture

- Packet ingestion flows through parser backends, flow tracking, and TCP stream reconstruction.
- Detection layers combine heuristics, fingerprinting, anomaly scoring, DPI carving, and project-aware context.
- Output surfaces include SQLite telemetry, markdown and JSON reports, Mermaid diagrams, and a FastAPI dashboard.

## Security and observability

The project treats explainability as part of detection quality. A finding is more useful when it includes the traffic path, why the rule fired, and where the operator should look next.

## Result

The project works as an end-to-end detection platform with offline PCAP replay, live capture, report artifacts, dashboard views, and GitHub-ready documentation.
