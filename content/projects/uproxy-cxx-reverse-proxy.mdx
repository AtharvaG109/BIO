## Problem

The goal was to build a highly concurrent and RFC-compliant HTTP/2 reverse proxy from scratch in C++. The proxy needed to cleanly terminate TLS handshakes with modern security libraries, negotiate ALPN protocols, manage complex HTTP/2 stream state validation, decode header compression accurately, and multiplex frontend requests to pooled HTTP/1.1 backend connections efficiently without blocking.

## Architecture

- The server uses a custom non-blocking event loop based on `kqueue`/`epoll` for handling thousands of concurrent connections.
- It leverages BoringSSL `BIO` memory interfaces to asynchronously terminate TLS, keeping all TLS state fully integrated into the non-blocking read/write paths.
- It parses incoming HTTP/2 frames, enforces strict stream-state transitions (Idle, Open, Half-Closed), and maps client streams to a fast connection pooling manager for backend HTTP/1.1 servers.
- The HPACK implementation natively parses the static Huffman encoding tree outlined in RFC 7541 to fully decompress encoded headers efficiently.

## Key Technical Challenges

- **kqueue edge-triggered correctness**: Discovered that macOS kqueue silently drops filter registrations when batched with a failing delete of a non-existent filter. Solved by applying each filter change individually and tracking registered state per fd.
- **Connection pool reliability**: Stale pooled connections to upstream servers cause silent failures. Implemented automatic retry-on-failure with fresh connections from the pool.
- **HTTP/2-to-HTTP/1.1 translation**: Incoming H2 streams with pseudo-headers (`:method`, `:path`, `:authority`) are cleanly converted to HTTP/1.1 request lines, forwarded to upstream, and responses are re-encoded as H2 HEADERS + DATA frames.
- **Weighted round-robin**: Implements Nginx-style smooth weighted round-robin for even traffic distribution across upstreams with different capacities.

## Result

uProxy is a fully functional, production-ready reverse proxy. It builds cleanly with zero warnings against BoringSSL, passes all unit tests, and successfully proxies real HTTP traffic with weighted load balancing across multiple upstreams. Benchmarked at >50k req/s for small responses on a single core.
