## Problem

The goal was to build a useful host monitor while staying precise about scope. The project traces activity, classifies it in userspace, and only claims kernel blocking for the subset of outbound IPv4 connects it can actually enforce.

## Architecture

- eBPF programs hook Linux tracepoints and stream structured events through a ring buffer.
- An async Rust userspace engine applies YAML policies, suppression, and deduplication.
- Optional cgroup connect4 enforcement blocks exact outbound IPv4 rules.

## Operational detail

The project keeps monitoring and enforcement separate. That makes demos more honest and makes it easier to explain which events are visible, which are classified, and which can be blocked.

## Result

Sysguard is a repo-ready prototype for hands-on systems security experimentation with readable output, sample policies, and a clear Linux-first workflow.
